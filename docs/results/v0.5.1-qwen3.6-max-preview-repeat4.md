This is the sanitized release-qualification result for PyFixAgent v0.5.1. It uses schema v2 generic tasks, isolated fixture copies, temporary Git repair worktrees, agent-visible tests, and external holdout tests. Holdout output is never supplied to the repair agent.

After model calls, `pyfixagent-benchmark --validate` rechecked every fixture with two hash seeds. All 15 remained isolated failing baselines with external holdouts. The benchmark runner also materialized exported patches as bytes and assigned each pytest command a fresh external base temporary directory, preventing Windows newline conversion and stale pytest temp-directory permissions from changing the result.

## Failure analysis

All eight failures passed their visible suites and failed only at the external holdout gate:

- `decimal_invoice` failed all four repetitions. The edit quantized values to cents but relied on `Decimal`'s default tie-breaking, so a positive half-cent subtotal rounded to an even cent instead of the domain-required upward cent.
- `file_extension` failed all four repetitions. The edit correctly replaced only the final suffix but did not normalize an extension argument that already contained a leading dot, producing a doubled separator.

No case-specific task hints or holdout feedback were added. The prompt already contained generic normalization checks, yet the model did not consistently turn them into code. This is evidence that adding more benchmark-specific prose is not a scalable fix; v0.6 should add an independent, holdout-blind semantic acceptance responsibility based on inferred contracts and counterexamples.

## Comparison and interpretation

The v0.4.0 `qwen3.6-flash` qualification reached 91.7% final success and 93.3% Pass@4 with 155,230 total tokens. v0.5.1 with `qwen3.6-max-preview` reached 86.7% final success and Pass@4 while using 207,392 tokens. These are small-model samples rather than a general model ranking, but they show that the larger model and thinking mode did not improve this benchmark's hidden-case generalization.

The 100% visible-test rate and zero policy violations qualify the transactional execution path, model integration, edit backends, and visible verification loop. The lower holdout rate prevents a stronger semantic-quality claim. PyFixAgent v0.5.1 should therefore be treated as a safer execution and retry baseline, not as an across-the-board repair-quality improvement over v0.4.0.

This is a small curated local benchmark, not a production-grade claim or a comparison with SWE-bench.
