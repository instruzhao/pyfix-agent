# v0.6.1 — Static Repository Context

v0.6.1 adds a bounded, execution-free repository understanding layer. Traceback evidence remains the seed signal; a static Python index can now expand those seeds through direct import relationships and select related symbol ranges under a deterministic source-context budget.

## What changed

- Added an AST-only repository index for Python files, top-level functions/classes, class methods, and direct import edges.
- Added content-addressed JSON caching outside the workspace, with automatic invalidation after edits.
- Added an in-memory fallback when the configured cache would be inside the workspace being repaired, preventing cache artifacts from entering candidate diffs.
- Added forward dependency and reverse importer traversal with bounded depth, stable scores, and deterministic ordering.
- Added symbol-range selection when an imported or query-referenced symbol can be identified; otherwise selection uses a bounded file prefix.
- Added `context.max_selected_tokens` and repository file, byte, depth, related-file, and snippet-line limits.
- Shared one repository service between repair context and semantic-review context while keeping `RepairEngine` independent from index implementation details.
- Added repository fingerprint, cache source, index counts, graph distance, score, selected symbol, token estimate, and budget-truncation fields to trace schema 1.3.
- Added `--repository-context` and `--no-repository-context` compatibility switches.

## Safety and scope

- Indexing reads and parses source; it never imports or executes project modules.
- Syntax and decoding failures are recorded per file and do not abort indexing.
- Paths still come from the existing workspace-bounded Python scanner.
- Repository context does not change edit authorization, test execution, rollback, semantic acceptance, or external-holdout isolation.
- This is a static Python import graph, not dynamic call-graph resolution, cross-language indexing, embeddings, or RAG.

## Compatibility

- The default CLI and benchmark configuration enable repository context.
- Direct `DefaultAgent` construction remains opt-in to preserve existing library behavior and trace shapes.
- `--no-repository-context` restores the v0.6.0 traceback/full selector behavior.
- Existing context fields remain present; schema 1.3 adds repository metadata only when the feature is enabled.

## Qualification

- Component tests cover execution-free parsing, imports and symbols, syntax tolerance, disk cache hits, content invalidation, inside-workspace cache bypass, deterministic graph expansion, token budgeting, trace metadata, and shared repair/review assembly.
- The full non-integration suite passes 168 tests, and all 15 benchmark fixtures remain isolated failing baselines with external holdouts.
- The 15-case `qwen3.6-max-preview` release run reached 100% visible, candidate-holdout, review-acceptance, and final success with zero false accepts or false rejects. See `docs/results/v0.6.1-qwen3.6-max-preview.md`.
