# v0.6.3 — Kimi Default and Provider-Safe Thinking

v0.6.3 changes the default model from `qwen3.6-max-preview` to the Alibaba Cloud deployment of `kimi-k2.6`. It also separates generic generation limits from provider-specific thinking controls so a model switch does not silently carry incompatible parameters into repair or review calls.

## What changed

- Changed `model.name` to `kimi-k2.6` while retaining the DashScope OpenAI-compatible endpoint and `DASHSCOPE_API_KEY` environment variable.
- Enabled Kimi thinking mode with temperature `1.0` and standard system-role prompting.
- Removed `thinking_budget` from the default repair and semantic-review configuration because that control is not supported by Kimi on this endpoint.
- Kept the reviewer output cap at 3,072 tokens and the repair output cap at 8,192 tokens, so both paths remain bounded even without a thinking-budget control.
- Changed reviewer configuration assembly so it never inherits a repair thinking budget. A reviewer budget is sent only when `semantic_review.thinking_budget` explicitly defines one.
- Preserved explicit thinking-budget support for compatible custom configurations such as Qwen deployments.

## Compatibility boundary

The repair engine, semantic reviewer, benchmark runner, and model client still receive the same generic interfaces. Provider-specific choices remain in YAML and CLI composition helpers:

- `enable_thinking` is optional and provider-dependent.
- `thinking_budget` is optional and emitted only when present for repair, or explicitly present in semantic-review configuration for review.
- `system_prompt_as_user` remains available for models that need a merged user-message contract.
- Historical qwen qualification reports remain unchanged and continue to describe the model configuration used for those runs.

## Qualification scope

v0.6.3 is completed and versioned locally. Its release gate is entirely offline: all 182 non-integration tests pass, one real-service integration test is deselected, and all 24 benchmark fixtures remain isolated failing baselines with external holdouts. Compilation, CLI version output, diff checks, and Git worktree checks also form part of the gate. No Kimi API call or new model-quality claim is part of this local release.

## Publication state

The v0.6.3 commit and annotated tag are local only. They are not pushed to GitHub, and the existing remote branch and tags are not changed.
