# v0.4.0 — Role-oriented Internal Architecture

v0.4.0 is primarily an architecture release. It addresses the growing coupling in the repair loop and benchmark runner without changing the public agent API, CLI commands, benchmark manifest schema, or trace schema.

## What changed

- `DefaultAgent` became a small compatibility and dependency-assembly facade.
- `RepairEngine` now coordinates an explicit, deterministic repair state machine.
- Visible tests, workspace setup, context selection, prompt construction, model calls, edit application, attempt evaluation, and retry decisions have independent owners.
- Patch and JSON replacement edits implement a shared backend boundary and can be tested independently.
- The benchmark runner is split into manifest, workspace, holdout, runner, metrics, reporting, and CLI modules.
- `pyfixagent.benchmark` remains the stable import and command facade.
- Component contracts and architecture-focused unit tests were added.
- Retry policy can switch from replacement to patch when exact source anchors are repeatedly unusable, and no-op replacements are rejected.
- Both edit prompts use one generic contract/self-counterexample check; no benchmark-specific task hints are used.
- Benchmark fixture copies exclude bytecode/cache artifacts, and protocol validation uses two deterministic hash seeds.

## Compatibility

- `DefaultAgent(...)` constructor and `run(task)` remain supported.
- `pyfixagent` and `pyfixagent-benchmark` commands are unchanged.
- benchmark manifest schema remains version 2.
- structured trace schema remains version 1.0.
- default model remains `openai/qwen3.6-flash` through the configured DashScope-compatible endpoint.

## Release qualification

The release gate consists of the complete non-integration test suite, benchmark protocol validation, CLI/version checks, and all 15 external-holdout benchmark cases repeated four times with the configured real model. The sanitized real-model result is recorded in `docs/results/v0.4.0-qwen3.6-flash-repeat4.md`.
