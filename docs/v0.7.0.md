# v0.7.0 — Execution Isolation

v0.7.0 moves pytest execution behind a backend-neutral sandbox contract and adds an optional hardened container path. The release keeps the local backend as the compatibility default, but the container backend is the intended boundary for unfamiliar projects.

## Container Execution

- Added a shared `Sandbox` protocol with command execution, workspace rebinding, pytest temporary-path mapping, and environment metadata.
- Added ephemeral Docker/Podman execution for CLI repairs, benchmark visible tests, and external holdouts.
- Requires a detached temporary Git worktree; container mode rejects `--in-place`.
- Mounts only the disposable worktree at `/workspace`.
- Defaults to no network, a read-only root, a bounded `noexec` tmpfs, non-root UID, all capabilities dropped, no-new-privileges, and CPU/memory/PID/wall-time limits.
- Names each container and force-removes it after a host timeout.

## Dependencies and Reproducibility

- Added `containers/Dockerfile` with a digest-pinned Python base image and a fully resolved `containers/requirements.lock` for the included Python/scientific fixtures.
- Added an `image_only` dependency policy and rejects runtime `pip` installation through the sandbox.
- Uses `--pull never` by default so a repair cannot fetch a replacement image implicitly.
- Trace schema 1.5 records the effective policy, runtime/server version, requested image, and resolved digest or ID when available.

## Patch Approval

- Replaced the exported raw `git apply` hint with `pyfixagent-apply`.
- The first invocation validates a clean Git workspace, patch format, edit policy, changed-file/line budgets, and `git apply --check`, then prints the SHA-256 and exits without mutation.
- A second invocation must provide the exact digest through `--approve`; changed patch content or workspace state invalidates approval.
- Applied changes remain uncommitted.

## Trace Audit and Viewer

- Added privacy auditing for residual absolute paths and unsafe source-bearing fields in safe-mode traces.
- Added `pyfixagent-trace-viewer`, which produces standalone HTML with escaped JSON, no scripts or remote assets, and a restrictive content-security policy.
- The viewer can apply path or safe redaction before rendering and can fail automation when audit findings remain.

## Compatibility

- Existing `LocalSandbox` behavior and `DefaultAgent` construction remain available.
- Existing trace fields remain intact; schema 1.5 only adds execution metadata.
- Benchmark report schema remains version 4.
- The configured default model remains `deepseek-v4-flash`; v0.7.0 introduces no model-specific prompts or business rules.

## Security Boundary

Container mode is defense in depth, not VM-grade isolation. It shares the host kernel and depends on Docker/Podman daemon security, image provenance, and platform bind-mount behavior. The local backend remains suitable only for trusted projects.

## Local Release Gate

The final gate with `RUN_DOCKER_TESTS=1` passes 205 tests with only the external LLM integration test skipped. The container test module passes all 9 tests, including the real Docker smoke test. All 24 benchmark fixtures validate as isolated failing baselines with external holdouts. Compilation, CLI/version checks, safe trace viewer generation and privacy audit, diff checks, and Git worktree cleanup also pass.

The Docker CLI and Desktop application are installed locally. The WSL runtime was updated during qualification, and a host restart allowed Docker Desktop 29.4.2 to reach `running`. Docker Hub authentication was unreachable from this network, so the identical official Docker Library `python:3.12-slim` base was obtained through AWS Public ECR, verified at digest `sha256:57cd7c3a7a273101a6485ba99423ee568157882804b1124b4dd04266317710de`, and pinned in the recipe. The resulting `pyfixagent-runner:0.7.0` image built successfully and executed the real sandbox smoke test.

The v0.7.0 commit and annotated tag are local only and are not pushed to GitHub.
