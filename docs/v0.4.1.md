# v0.4.1 — CI and Benchmark Packaging Maintenance

v0.4.1 is a maintenance release for the v0.4 architecture baseline. It fixes the release workflow and makes the dependencies required by the complete benchmark protocol explicit. Repair behavior, public APIs, configuration, benchmark schema, and trace schema are unchanged.

## What changed

- Added the optional `benchmark` dependency group with `matplotlib`, `numpy`, and `scikit-learn`.
- Kept the supported Python 3.10–3.13 unit-test matrix focused on non-integration tests and installed command checks.
- Moved the complete benchmark fixture/holdout validation into one dedicated Python 3.13 job.
- Limited automatic workflow runs to pushes on `main` and pull requests targeting `main`, so version-tag synchronization does not duplicate CI runs.
- Added per-ref workflow concurrency with cancellation of superseded runs.

## Compatibility

- `DefaultAgent(...)`, `RepairEngine`, `pyfixagent`, and `pyfixagent-benchmark` remain compatible with v0.4.0.
- The benchmark manifest remains schema version 2.
- Structured traces remain schema version 1.0.
- The default configured model remains `openai/qwen3.6-flash`.
- Base installation remains `python -m pip install -e .`; complete benchmark validation uses `python -m pip install -e ".[benchmark]"`.

## Qualification

- The complete non-integration suite contains 129 passing tests with one real-model integration test deselected.
- All 15 benchmark baselines validate as isolated, intentionally failing fixtures with external holdouts.
- GitHub Actions validates Python 3.10, 3.11, 3.12, and 3.13 plus one dedicated benchmark protocol job.
- Because no repair, prompt, context, or model behavior changed, the v0.4.0 four-repetition real-model report remains the applicable behavioral qualification baseline.
