# v0.5.1 — Failure-delta Semantic Retry

v0.5.1 builds semantic retry behavior on the transactional worktree boundary introduced in v0.5.0. Retry decisions use normalized test outcomes and failure deltas; they do not inspect benchmark holdouts or add project-specific task hints.

## What changed

- Extended `RetryDecision` with explicit rollback, checkpoint, and context-expansion actions.
- Moved regression/no-progress/partial-progress classification into `RetryPolicy`.
- Rolled back no-progress attempts as well as regressions when temporary-worktree execution is active.
- Kept partial improvements as checkpoints for incremental repair.
- Added `ContextExpansionPolicy` as an independent bounded strategy.
- Doubled traceback line/file limits at expansion level 1 and allowed full context at level 2 when configured.
- Added semantic retry feedback containing fixed, remaining, and new failure IDs plus the actual rollback/checkpoint state.
- Recorded retry reasons, workspace actions, effective context limits, and expansion levels in trace schema 1.1.
- Made the maximum expansion level configurable through `context.max_expansion_level`.
- Isolated every pytest command in a fresh external base temporary directory and materialized exported patches without newline translation.

## Default model

The default model is `openai/qwen3.6-max-preview` through the configured DashScope OpenAI-compatible endpoint. Thinking mode is enabled with a 4096-token budget, and the output contract is merged into the user message for this model family. Model qualification uses generic schema v2 benchmark tasks and external holdouts.

## Compatibility

- Patch/replacement parsing and backend switching remain independent from semantic retry.
- In-place library compatibility mode cannot safely roll back an iteration; decisions are still traced, but the actual workspace action records that the edit was kept in place.
- Setting `context.max_expansion_level` to `0` disables context growth while retaining normalized retry decisions.
- Benchmark manifest schema remains version 2 and trace schema remains version 1.1.

## Qualification

- Component tests cover every semantic retry decision and both context-expansion stages.
- End-to-end tests verify no-progress rollback, expanded next-attempt context, aggregate patch correctness, and an unchanged original checkout.
- All 15 benchmark fixtures were revalidated as isolated failing baselines after the real-model run; no repaired fixture was retained.
- The four-repetition real-model qualification completed 60 runs with 100% visible-test success and 86.7% external-holdout/final success. Thirteen cases passed 4/4; `decimal_invoice` and `file_extension` passed visible tests but failed holdouts 0/4.
- The result is intentionally not repaired with case-specific prompt hints. It identifies post-visible semantic acceptance as the next architectural gap rather than hiding it with benchmark answers.
