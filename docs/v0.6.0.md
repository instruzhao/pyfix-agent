# v0.6.0 — Semantic Acceptance

v0.6.0 separates a visible-test candidate from a finally accepted repair. A candidate that passes configured pytest commands is checkpointed and reviewed by an independent, holdout-blind semantic reviewer before the run can report success.

## What changed

- Added strict review contracts, JSON parsing, evidence-path validation, and a pure `ReviewPolicy`.
- Added changed-file and visible-test review context that cannot access external benchmark holdouts.
- Added deterministic structural risk cues for delimiter composition and implicit numeric tie-breaking.
- Promoted evidence-based warnings that match code-derived cues into bounded semantic revisions.
- Added fail-closed `needs_review` behavior while still exporting the last visible-pass candidate patch.
- Added separate `visible_success`, `acceptance_status`, candidate patch, review records, and semantic revision counts.
- Upgraded trace schema to 1.2 and benchmark report schema to 3.
- Added false-accept, false-reject, review-token, review-count, and candidate-success metrics.
- Added CLI switches for enabling or disabling semantic review and exit code 2 for `needs_review`.

## Safety and isolation

- The reviewer never edits files; repair backends remain the only edit boundary.
- Raw reviewer output is never inserted directly into a repair prompt. Only validated risk IDs, reasons, and counterexample properties are rendered by `ReviewFeedbackBuilder`.
- External holdouts are evaluated only after the Agent closes its isolated repair worktree.
- A rejected candidate can be evaluated by the benchmark runner, but its holdout output never returns to the Agent.
- Semantic review is bounded by the existing edit limit and at most two configured semantic revisions.

## Compatibility

- `DefaultAgent` library construction keeps semantic review opt-in for existing direct callers.
- The default CLI and benchmark configuration enable semantic review.
- `--no-semantic-review` preserves v0.5.1 visible-test acceptance behavior.
- Review-disabled workspace actions and benchmark failure names remain backward compatible.

## Qualification

- Unit and end-to-end tests cover accept, revise, abstain/error, deterministic cue promotion, candidate export, worktree preservation, and external false-reject evaluation.
- Real-model smoke tests repaired both v0.5.1 stable holdout failures without case-specific task hints: `file_extension` normalized an already-marked suffix, while `decimal_invoice` selected explicit half-up rounding and a Decimal empty-sum identity.
- The 15-case `qwen3.6-max-preview` release run reached 100% visible, candidate-holdout, review-acceptance, and final success with zero false accepts or false rejects. See `docs/results/v0.6.0-qwen3.6-max-preview.md`.
- Full-suite and benchmark-baseline checks are completed before the annotated release tag is created.
