# v0.5.0 — Transactional Workspaces

v0.5.0 changes the default CLI repair boundary from in-place edits to a detached temporary Git worktree. The selected checkout is inspected but not modified; successful repairs are exported as aggregate patches for review and explicit application.

## What changed

- Added `WorkspaceTransaction` as the sole owner of temporary worktree creation, checkpoints, rollback, patch export, and cleanup.
- Added a checkpoint after accepted partial or successful iterations.
- Added automatic rollback when an iteration introduces new failing tests.
- Added `final_patch_path` and a directly usable `final_patch_command` to structured results.
- Added schema 1.1 trace fields for workspace actions and retry reasons.
- Added configurable ordered pytest commands as argv lists.
- Added `TestCommandPolicy`, which rejects shell strings, shell operators, and non-pytest commands.
- Updated the benchmark runner to materialize the exported patch only inside its disposable outer workspace before external holdout evaluation.
- Changed the default configured model to `openai/qwen3.6-max-preview` through the DashScope-compatible endpoint.

## Compatibility

- `DefaultAgent` remains source-compatible and keeps in-place execution as its library-level default.
- The user-facing CLI enables temporary-worktree isolation by default.
- `--in-place` explicitly requests the earlier mutation behavior.
- `--allow-dirty` still bypasses the clean-workspace guard, but the isolated repair starts from the recorded HEAD revision.
- Repair modes, benchmark manifest schema 2, and existing trace 1.0 readers remain compatible with additive trace 1.1 fields.

## Qualification

- Unit and component tests cover nested workspaces, checkpoint persistence, regression rollback, final patch export, cleanup, and test-command rejection.
- The complete benchmark protocol continues to require intentionally failing visible baselines and external holdouts.
- v0.5.1 will build semantic no-progress retry and context expansion on top of this transaction boundary.
